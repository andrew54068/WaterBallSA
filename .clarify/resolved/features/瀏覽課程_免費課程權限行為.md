# 釐清問題

關於「免費課程 (Free Curriculum)」的定義與行為，目前規格中僅提及 `GET /curriculums/free` endpoint。請問免費課程是否需要使用者「購買（0 元購買）」後才能觀看，還是可以直接觀看？

# 定位

Feature：瀏覽課程.feature
Rule：查詢免費課程列表

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 需走購買流程 (0 元訂單)，產生 Purchase 紀錄後才能觀看 |
| B | 不需購買，任何登入使用者皆可直接觀看內容 |
| C | 不需購買，連未登入訪客皆可直接觀看內容 |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- 購買課程.feature (0 元購買邏輯)
- 瀏覽單元.feature (Access Control Rule)
- 權限驗證 Middleware

# 優先級

High

---
# 解決記錄

- **回答**：B - 不需購買，任何登入使用者皆可直接觀看內容
- **更新的規格檔**：spec/features/瀏覽單元.feature
- **變更內容**：
  1. 完善「若單元未設為公開，使用者必須擁有該課程才能觀看」的 Examples (區分已購/未購)。
  2. 新增規則「使用者已登入即可觀看免費課程的單元 (無需購買)」與 Examples，明確定義免費課程只需登入即可存取，不需走購買流程。
