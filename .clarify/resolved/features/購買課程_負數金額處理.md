# 釐清問題

關於「折扣後價格」計算，若優惠券折扣後金額變為負數 (e.g. 課程 300, 折抵 500)，之前的規則是「驗證失敗 (拒絕購買)」。是否考慮改為「以 0 元結帳」？

# 定位

Feature: 購買課程
Prior Rule: 優惠券折抵金額不可大於課程價格

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 維持現狀 (Reject)：金額小於 0 視為錯誤，必須湊到足以折抵或換小額 Coupon |
| B | 允許 0 元 (Cap at Zero)：若折抵超過價格，則視為 0 元 (Free Purchase) |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- Purchase Calculation Logic
- User Experience

# 優先級

Low

---
# 解決記錄

- **回答**：A - 維持現狀 (Reject)：金額小於 0 視為錯誤，必須湊到足以折抵或換小額 Coupon
- **更新的規格檔**：無 (維持原規格)
- **變更內容**：
  1. 確認維持 `優惠券折抵金額不可大於課程價格` 的限制規則。
