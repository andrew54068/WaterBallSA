# 釐清問題

關於「訂單狀態 (Purchase Status)」，ERM 定義了 `PENDING` 與 `COMPLETED`。但在 Mock Payment 且立即開通的流程中，是否真的需要 `PENDING` 狀態？(還是建立即完成？)

# 定位

ERM: Table Purchase
Feature: 購買課程

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 是，保留 PENDING：建立訂單與付款確認是兩步，即使是 Mock 也有時間差 |
| B | 否，僅 COMPLETED：Mock Payment 是同步的，訂單建立即完成，簡化狀態機 |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- Purchase State Machine
- API Flow

# 優先級

Low

---
# 解決記錄

- **回答**：B - 標準流程 (Standard)：保留 `PENDING`，建立訂單與確認付款分為兩步驟
- **更新的規格檔**：spec/features/購買課程.feature
- **變更內容**：
  1. 明確定義訂單生命週期：建立 (PENDING) -> 付款 (COMPLETED)。
  2. 即使是 Mock Payment，仍需維持此狀態轉換，以保持架構的可擴展性 (未來接真實金流)。
