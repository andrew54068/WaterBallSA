# 釐清問題

關於「優惠碼驗證 (Coupon Validation)」，針對新加入的屬性（有效期限、使用次數上限），若驗證失敗，系統應該回傳明確的錯誤原因，還是統一模糊回應？

# 定位

Feature：購買課程 (Purchase Curriculums)
Rule：驗證優惠碼有效性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 嚴格模式 (Explicit)：回傳具體錯誤訊息，如 "優惠券已過期" 或 "已達使用上限" |
| B | 模糊模式 (Vague)：統一僅回傳 "優惠券無效"，不透露具體原因（防止猜測規則）|
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- 購買課程.feature (Validation Examples)
- API Error Codes

# 優先級

Medium
