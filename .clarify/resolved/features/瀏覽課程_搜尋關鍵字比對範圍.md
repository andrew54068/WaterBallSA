# 釐清問題

課程搜尋功能 (Search) 的關鍵字比對規則為何？是僅比對標題 (Title)，還是也要比對說明或其他欄位？

# 定位

Feature：瀏覽課程.feature
Rule：搜尋課程支援關鍵字、分頁與排序

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 僅比對課程標題 (Title) |
| B | 比對標題 (Title) 與 說明 (Description) |
| C | Full-text Search (標題、說明、章節名稱) |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- 瀏覽課程.feature (Search Example)
- Database Query Performance (Index design)

# 優先級

Medium

---
# 解決記錄

- **回答**：B - 比對標題 (Title) 與 說明 (Description)
- **更新的規格檔**：spec/erm.dbml, spec/features/瀏覽課程.feature
- **變更內容**：
  1. `erm.dbml`: 在 Curriculum 實體中補上 `description` 屬性（因為要支援搜尋說明，隱含必須有此欄位）。
  2. `瀏覽課程.feature`: 明確定義搜尋規則為「比對標題與說明」，並加入相關 Examples 驗證兩個欄位皆可觸發關鍵字命中。
