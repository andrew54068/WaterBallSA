# 釐清問題

關於「無管理員角色 (No Admin Role)」的決策，既然系統內無 Admin 帳號，請問「發布課程 (Toggle Published)」、「建立課程」等後台操作應如何執行？

# 定位

ERM: Table Curriculum (published field)
Feature: 瀏覽課程

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 僅限資料庫操作 (DB Access)：管理者需直接連線到 DB 修改 `published` 欄位 (MVP) |
| B | 隱藏 API (Secret Key)：透過帶著特定 Secret Header 的 API 呼叫來操作 (如 CI/CD 或 Script) |
| C | 特定 User ID：Hardcode 特定 Google User ID 為管理員 |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- API Security
- Operational Workflow

# 優先級

Medium

---
# 解決記錄

- **回答**：A - 僅限資料庫操作 (DB Access)：管理者需直接連線到 DB 修改 `published` 欄位 (MVP)
- **更新的規格檔**：無 (確認不需新增 Admin API 或 Role)
- **變更內容**：
  1. 確認後台管理操作依賴 DB 直接存取。
  2. 規格中不需要定義「建立課程」、「發布課程」的 API。
