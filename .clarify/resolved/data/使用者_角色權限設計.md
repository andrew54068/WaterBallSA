# 釐清問題

關於「使用者角色 (User Roles)」，目前系統是否區分「管理員 (Admin)」與「一般學員 (Student)」？spec 提到「未發布課程對一般使用者隱藏」，暗示有非一般使用者（如管理員）可見。

# 定位

ERM: Table User
Feature: 瀏覽課程 (Visibility)

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 單一角色：系統僅有一般學員，管理者直接操作 DB 或其他後台 (MVP) |
| B | 簡單角色欄位：User Table 新增 `role` (ADMIN, STUDENT) |
| C | RBAC：新增 Role/Permission Tables (複雜度高) |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- ERM (User fields)
- Auth Middleware
- API Visibility Logic

# 優先級

Medium

---
# 解決記錄

- **回答**：A - 單一角色模式 (No Role)：系統僅有一般學員，管理者直接操作資料庫 (MVP 簡化)
- **更新的規格檔**：無顯著變更 (ERM 維持原樣)
- **變更內容**：
  1. 確認 `User` Table **不需新增** `role` 欄位。
  2. 隱含規則：由於系統內無 Admin 角色，"未發布的課程" 對所有透過 API 存取的使用者皆不可見 (僅 DB 管理者可見)。
